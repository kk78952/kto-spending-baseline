# Claude Code — KTO Spending Baseline

## Project context

This is a **data pipeline + static dashboard** project. It is NOT a chatbot or API service.

- Pipeline: Python scripts in `pipeline/` that query Trino and run ETS forecasting
- Dashboard: React static site in `dashboard/` reading JSON/CSV from `data/`
- No web server to run — serve dashboard with `python -m http.server` from project root

## Key rules

1. **Never commit credentials** — `config/*.env` is gitignored
2. **`price / 100` = actual VND** — the `price` column in `recharge_deliver` is VND × 100
3. **Exclude bigtypes 21, 37–43** from spending queries (player transfers, not real spending)
4. **GS1 is a department umbrella** — never add KTO figures to GS1 aggregates
5. **`data/` is gitignored** — generated by pipeline, do not commit CSVs/JSONs

## Trino connection

- Host: 10.60.34.154:8443, catalog: hive, schema: kto_658
- Credentials in `config/credentials.env` (gitignored)

## Pipeline order (run_pipeline.sh)

1. `parse_event_calendar.py` — xlsx → events.json
2. `fetch_daily.py` — SQL → CSV
3. `build_baseline.py` — ETS training + forecast
4. `generate_alerts.py` — pattern rules → alerts.json

## SQL conventions

- Always filter `ds = :target_date` (partition pruning)
- Filter `moneytype = 'Gold'` on both moneychange tables
- Use `MIN_BY(before, time)` for BOD balance per role
- Cohort CASE WHEN: `viplevel >= 12 → whale, >= 7 → dolphin, else minnow`

## Dashboard data contract

`data/daily_metrics.csv` columns:
```
ds, cohort, total_gold_spent, active_spenders, new_spenders, returning_spenders,
avg_spend_per_role, balance_velocity, avg_balance_eod, total_gold_received,
gold_from_recharge, total_vnd_recharged, event_flag, baseline_p50,
baseline_upper, baseline_lower, signal
```

`data/forecast.json` structure:
```json
{ "generated_at": "YYYY-MM-DD", "cohorts": { "whale": [...], "dolphin": [...], "minnow": [...] } }
```

Each cohort array: `[{ "date", "forecast", "lower", "upper", "confidence", "is_event" }]`

`data/alerts.json`: `{ "generated_at", "as_of_date", "alerts": [{ "type", "cohort", "title", "message" }] }`
