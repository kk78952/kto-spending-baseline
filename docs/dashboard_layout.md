# Dashboard Layout — Section Descriptions

Reference: `project_spec.md §9` and `dashboard/app.jsx`

---

## Header

- Game tag: KTO | Kim Thánh Online
- Title: "Daily spending baseline"
- Sub: Report date (today), Data date (yesterday D-1), Model name

---

## Section 1 — Yesterday's health check

**Purpose:** Instant go/no-go on economy health.

**4 metric cards:**
| Card | Value | Sub-text |
|---|---|---|
| Actual spend | SUM(total_gold_spent) all cohorts | "All cohorts · Gold" |
| Baseline (P50) | SUM(baseline_p50) all cohorts | "ETS predicted" |
| Deviation | (actual − baseline) / baseline × 100 | "Within ±15% band" or "Outside band" |
| Spender count | SUM(active_spenders) | "Active Gold spenders" |

**Color logic on deviation:**
- Green (`#5DCAA5`): |dev| < 15%
- Amber (`#EF9F27`): 15% ≤ |dev| < 25%
- Red (`#F0997B`): |dev| ≥ 25%

---

## Section 2 — Cohort breakdown

**Purpose:** Diagnose which segment is responsible for deviation.

**Table columns:** Cohort | Spend | Velocity | Avg/role | Spenders (+new) | vs Base

**One row per cohort** (Whale → Dolphin → Minnow).

**Color rules:**
- Velocity < 0.40 → coral (whale hoarding signal)
- vs Base: green if positive, coral if < −10%, grey otherwise

---

## Section 3 — 30-day trend chart

**Purpose:** Visual context — is today normal vs recent history?

**SVG line chart (no library):**
- Solid blue line: actual spend (aggregated across cohorts)
- Dashed line: ETS baseline (P50)
- Grey fill: confidence band (P25–P75, ±15%)
- Green dots: days where actual > P75 (over-baseline)
- Coral dots: days where actual < P25 (under-baseline)
- Amber fill region: event periods
- X-axis: date labels every 5 days

---

## Section 4 — Velocity & spender trends (7-day sparklines)

**Purpose:** Early signals from trend direction.

**Layout:** 3 rows (one per cohort), each with 2 side-by-side cards.

Left card (velocity):
- Cohort label + "velocity" tag
- Large number: current velocity value
- Trend badge: "declining" (cohort color) or "stable" (grey)
- Sparkline

Right card (spenders):
- Cohort label + "spenders" tag
- Large number: active spender count
- Small green "+N new" badge
- Sparkline (dimmed color)

## Section 5 — Gold sinks by shop

**Purpose:** Understand WHERE Gold is being spent.

**Source:** `data/shop_breakdown.csv` (generated by `sql/shop_breakdown.sql`)

**Horizontal bar chart:** logway group names, bar width proportional to spend, Gold amount + % on right.

**Note:** Player transfers (MarketStall, Trade) are excluded before aggregation.

---

## Section 6 — Alerts & signals

**Purpose:** Surface actionable patterns automatically.

**Alert card anatomy:**
- Icon circle (! warn / ✓ success / i info)
- Cohort badge (optional, colored)
- Bold title + descriptive message

**Alert types:** `warning` (coral), `success` (teal), `info` (blue)

**Rules evaluated** (from `generate_alerts.py`):
1. Churn: spend↓ + velocity stable + spenders↓
2. Hoarding: spend↓ + velocity↓ + spenders stable
3. Reserve burn: spend stable + recharge↓
4. UA working: spend↑ + velocity↑ + new_spenders↑
5. Unknown spike: actual > P75, no event flag
6. Content gap: actual < P25, no event flag
7. Whale hoarding: velocity < 0.40 for 3+ consecutive days

---

## Section 7 — 7-day forecast

**Purpose:** Forward planning and event sizing.

**Table columns:** Date | Forecast | Lower | Upper | Conf.

**Confidence:** ±15% day 1 → widens by ~3% per day.
Conf. column turns coral when beyond day 5 (confidence > ±25%).

Event rows highlighted with amber background and "EVENT" badge.
