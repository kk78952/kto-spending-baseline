# KTO Daily Spending Baseline

Daily dashboard that predicts KTO (Kim Thánh Online) Gold spending per day using Exponential Triple Smoothing (Holt-Winters), then compares actual vs baseline to detect anomalies, evaluate events, and forecast 7 days ahead.

## Quick start

### 1. Configure credentials

```bash
cp .env.example config/trino.env
cp .env.example config/credentials.env
# Edit both files with your Trino host/user/password
```

### 2. Install Python deps

```bash
pip install trino pandas statsmodels openpyxl
```

### 3. Run the pipeline

```bash
bash pipeline/run_pipeline.sh           # D-1 (yesterday)
bash pipeline/run_pipeline.sh 2026-04-09  # specific date
```

This runs 4 steps:
1. Parse `event_calendar/KTO_Nội_Dung_Vận_Hành.xlsx` → `event_calendar/events.json`
2. Fetch metrics from Trino → `data/daily_metrics.csv`, `data/velocity_history.csv`
3. Train ETS model → `data/forecast.json` (+ updates baseline columns in CSV)
4. Generate alerts → `data/alerts.json`

### 4. View the dashboard

```bash
cd dashboard
python -m http.server 8080
# Open http://localhost:8080
```

The dashboard reads from `/data/` at runtime. Serve from the project root, not the `dashboard/` subfolder:

```bash
# From project root:
python -m http.server 8080
# Open http://localhost:8080/dashboard/
```

## Data sources

All data in **Trino/Iceberg**, catalog `hive`, schema `kto_658`:

| Table | Purpose |
|---|---|
| `moneychange_reduce` | Gold outflows (spending) |
| `moneychange_add` | Gold inflows (recharge, rewards) |
| `recharge_deliver` | VND → Gold top-up events |

Key rule: `price / 100` = actual VND (field stores VND × 100).

## Cohorts

| Cohort | VIP level | ~Players | ~Revenue |
|---|---|---|---|
| Whale | 12+ | 1.1% | 47.5% |
| Dolphin | 7–11 | 10.8% | 39.5% |
| Minnow | 0–6 | 88.1% | 13.0% |

## ETS model

- Holt-Winters, additive trend + additive weekly seasonality
- Min 56 days training, recommended 180 days
- Event days excluded from training (±3/+5 day window)
- Forecast horizon: 7 days reliable (~15% MAPE), 14 days max

## Deployment (Netlify)

1. The `data/` folder is gitignored. Before deploying, either:
   - Copy `data/` into the Netlify publish directory manually, or
   - Use `netlify deploy` CLI after running the pipeline
2. Set publish directory to root (`.`) in Netlify settings
3. Set the redirect: `/ → /dashboard/index.html`

See `netlify.toml` for configuration.
