# Event Calendar

## Source file

`KTO_Nội_Dung_Vận_Hành.xlsx` — maintained by the KTO ops team. Updated regularly; re-parse on every pipeline run.

## Structure

Gantt-style layout across sheets named by year (2023, 2024, 2025, 2026):

| Row | Content |
|-----|---------|
| Row 1 | Dates across columns (starting column C, datetime objects) |
| Row 2 | Day-of-week numbers |
| Rows 3+ | Event tracks — col A = "Plan vận hành", col B = track number |

Event names appear at their start column and span across their duration. End date is determined by scanning rightward until the next non-null cell in the same row.

## Parsed output

`events.json` — generated by `pipeline/parse_event_calendar.py` on each run.

```json
[
  {
    "start_date": "2026-01-07",
    "end_date": "2026-01-20",
    "event_name": "TNTB",
    "track": 2,
    "year": 2026
  }
]
```

Revenue notes in event names (e.g. `"TNTB (2.5B, 3.1K PU)"`) are preserved in `event_name` as-is. The pipeline ignores the REV sheet.

## Event model integration

The ETS model excludes 3 days before + 5 days after each event from training. For forecast days with scheduled events, an `event_multiplier` is applied (estimated from historical event lift).

## Manual update

To add events manually (e.g. if xlsx is unavailable), edit `events.json` directly following the schema above. The pipeline will use `events.json` if it exists and is newer than the xlsx.
